# UKLidar Dataset for Superpoint Transformer

Integration of UK colourised LiDAR point clouds (EA National LIDAR Programme, Bluesky, MLS, etc.) with the [Superpoint Transformer](https://github.com/drprojects/superpoint_transformer) architecture for semantic segmentation.

Uses DALES-compatible class definitions (8 classes) so you can run inference with a pretrained DALES checkpoint out of the box.

## Classes

| ID | Class       | ASPRS codes mapped |
|----|-------------|--------------------|
| 0  | Ground      | 2                  |
| 1  | Vegetation  | 3, 4, 5            |
| 2  | Cars        | тАФ                  |
| 3  | Trucks      | тАФ                  |
| 4  | Power lines | тАФ                  |
| 5  | Fences      | тАФ                  |
| 6  | Poles       | тАФ                  |
| 7  | Buildings   | 6                  |

All other ASPRS codes (0, 1, 7, 9, 17, 18, 20) are mapped to **void** and excluded from metrics.

## Prerequisites

Install either PDAL or laspy for reading LAS/LAZ files:

```bash
# Option A: PDAL (recommended, usually already in the spt conda env)
conda install -c conda-forge python-pdal

# Option B: laspy with laszip support
pip install laspy[laszip]
```

## Command-line usage

### Preprocessing

Preprocessing builds the superpoint graph hierarchy (voxelisation, KNN, partition, etc.) and saves the result to `processed/`. This only needs to run once per dataset configuration.

```bash
python src/train.py \
    experiment=semantic/uklidar \
    datamodule.data_dir=/path/to/data \
    trainer.max_epochs=0
```

### Inference with a DALES checkpoint

```bash
python src/eval.py \
    experiment=semantic/uklidar \
    datamodule.data_dir=/path/to/data \
    ckpt_path=/path/to/dales_checkpoint.ckpt
```

### Training from scratch

If you have labelled UK LiDAR tiles in `raw/train/` and `raw/val/`:

```bash
python src/train.py \
    experiment=semantic/uklidar \
    datamodule.data_dir=/path/to/data \
    datamodule.trainval=False \
    datamodule.val_on_test=False \
    trainer.max_epochs=400
```

### Useful overrides

```bash
# Lower voxel resolution for sparse data (~2 pts/m┬▓)
datamodule.voxel=0.25

# Change tiling for very large tiles (e.g. 1km ├Ч 1km)
datamodule.xy_tiling=3

# Use a different model size
model=semantic/spt-2.yaml    # smaller
model=semantic/spt-64.yaml   # larger
```

## Python / Notebook usage

### Reading a single tile

```python
from src.datasets.uklidar import read_uklidar_tile

# Read a LAS/LAZ file into an SPT Data object
data = read_uklidar_tile("data/uklidar/raw/test/my_tile.laz")

print(data)
# Data(pos=[N, 3], rgb=[N, 3], intensity=[N, 1], y=[N])

print(data.pos.shape)       # torch.Size([N, 3])
print(data.rgb.shape)       # torch.Size([N, 3])  тАФ normalised to [0, 1]
print(data.intensity.shape)  # torch.Size([N, 1])  тАФ normalised to [0, 1]
print(data.y.shape)          # torch.Size([N])     тАФ semantic labels
```

### Reading with all labels set to void (no ground truth)

```python
data = read_uklidar_tile(
    "data/uklidar/raw/test/my_tile.laz",
    label_all_void=True,
)
# All labels will be 8 (void), useful for pure inference
```

### Instantiating the dataset via Hydra

```python
import hydra
from omegaconf import OmegaConf

# Load the config
with hydra.initialize(config_path="../configs"):
    cfg = hydra.compose(
        config_name="config",
        overrides=[
            "experiment=semantic/uklidar",
            "datamodule.data_dir=/path/to/data",
        ],
    )

# Instantiate the datamodule
datamodule = hydra.utils.instantiate(cfg.datamodule)

# Run preprocessing (only needed once)
datamodule.prepare_data()

# Set up datasets for a given stage
datamodule.setup(stage="test")

# Access the test dataset
test_dataset = datamodule.test_dataset
print(f"Number of test clouds: {len(test_dataset)}")

# Load a single processed cloud
nag = test_dataset[0]
print(nag)
```

### Running evaluation in a notebook

```python
import pytorch_lightning as pl
from src.models.semantic import SemanticSegmentationModule

# Load trained model
model = SemanticSegmentationModule.load_from_checkpoint(
    "/path/to/dales_checkpoint.ckpt"
)

# Set up datamodule (as above)
datamodule.setup(stage="test")

# Run evaluation
trainer = pl.Trainer(accelerator="gpu", devices=1)
results = trainer.test(model, datamodule=datamodule)
print(results)
```

## Tuning for your data

UK LiDAR data varies widely in point density:

- **EA ALS data** (~2тАУ20 pts/m┬▓): consider `voxel=0.25` or `voxel=0.5`
- **Bluesky colourised** (~20тАУ50 pts/m┬▓): `voxel=0.1` (default) works well
- **MLS data** (100+ pts/m┬▓): `voxel=0.05` and increase `xy_tiling` for memory

The preprocessing parameters in `uklidar.yaml` mirror the DALES defaults. For best results on your specific data, experiment with:

- `pcp_regularization` тАФ controls superpoint granularity (lower = finer)
- `knn` / `knn_r` тАФ neighbourhood size for geometric feature computation
- `ground_threshold` тАФ height threshold for ground elevation estimation
